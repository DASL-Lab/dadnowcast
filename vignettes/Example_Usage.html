The basic usage of the package involves the “nowcast” function, which fits a single nowcasting model to the data. The formula interface is used to specify the response variable and covariates, and lags can be specified using the “lag” function. The function converts the data into a standard format across all models so that a single “dadnow” object can be used to store the results of multiple models, and the same plotting and evaluation functions can be used regardless of the underlying model.

There’s also an autoplot() method for “dadnow” objects, which plots the nowcast against the observed data. By default, it plots all training data in the “dadnow” object, but the last_n argument can be used to specify how many of the most recent observations to plot.

dadnow2 <- nowcast(
  formula = dad ~ lag(cnisp, 3) + lag(ptsos, 3),
  data = df, 
  model = "ar", 
  date_col = "date", 
  params = list(order = 1)
)

dadnow2 <- add_model(dadnow2, model = "ar", params = list(order = 2)) |>
  add_model(model = "lm") |>
  add_model(model = "arx", params = list(order = 1)) |>
  add_model(model = "gam", params = list(smooths = list("cnisp_lag0"=10))) |>
  add_model(model = "rf", formula = dad ~ lag(cnisp, 2), params = list(ntree = 100, mtry = 3))

The package includes a “mechanistic” model, which can be seen in vignette("mechanistic_model"). The model takes the parameters sc and sp and requires a method (“normal”, “poisson”, or “negbinom”) to specify the distribution of the error term. Because it works slightly differently than the other models, it is added to a “dadnow” object using the add_mechanistic function, which takes the same formula interface as nowcast but with different parameters. The mechanistic model can be added alongside the other models, and the same plotting and evaluation functions can be used.

Note that the mechanistic assumption is sensitive to the values of sc and sp, and there will often be warnings about lack of convergence in the optimization process.

mech <- nowcast_mechanistic(
  dad ~ cnisp + ptsos + npos, df, date_col = "date",
  params = list(sc = 0.2, sp = 0.3, method = "poisson")
)

In addition to add_model() and add_mechanistic(), which adds a single model to a “dadnow” object, there is also a combine_dadnow function, which combines two “dadnow” objects.

dadnow3 <- combine_dadnow(dadnow2, mech) |>
  add_mechanistic(formula = dad ~ cnisp + ptsos + npos, params = list(sc = 0.2, sp = 0.3, method = "normal")) |>
  add_mechanistic(formula = dad ~ cnisp + ptsos + npos, params = list(sc = 0.2, sp = 0.3, method = "poisson"))

The add_model() interface allows for new formulas to be specified when adding new models.

dadnow4 <- add_model(dadnow3, formula = dad ~ lag(npos, 5) + lag(cnisp, 5) + lag(ptsos, 5), model = "xgboost", params = list(nrounds = 50)) |>
  add_model(formula = dad ~ lag(npos, 5) + lag(cnisp, 5) + lag(ptsos, 5), model = "lm") |>
  add_model(formula = dad ~ lag(npos, 5) + lag(cnisp, 5) + lag(ptsos, 5), model = "ar", params = list(order = 1))
